[Python] 'is' 와 '==' 의 차이점

헷갈릴수있는 비교연산자

2022-04-26

기존에 무언가를 비교할때 is 나 == 을 똑같다 생각하고 많이 쓰고는 했다.

하지만 똑같지 않았다.

먼저 결론은,

is 는 객체를 확인

== 는 값을 확인

>>> a = 1
>>> a is 1
True
>>> a == 1
True

위의 결과는 당연하다 1 은 boolean 값으로 True 이다.

반면,

>>> a = 1995
>>> a is 1995
False
>>> a == 1995
True

a 라는 변수에 1995 를 넣었을떄는 False 를 반환한다.

is 는 객체를 비교하고 == 는 값을 비교하기 때문이다.

>>> id(a)
140307053806736
>>> id(257)
140307053806576

위와 같이 서로의 메모리 주소도 다른것을 볼 수 있다.

기본중에 기본인것을 모르고 지금까지 하고있었다. 이와같이 is 와 == 을 헷갈리지말자.

RESTful API

REST 하다? 우선 REST 에 대해서 알아야 합니다. REST 란 웹에 존재하는 자원에 대해 고유한 URI 부여하는 방법입니다. 현재 가장 널리 알려져 사용되고있는 기술이기도 합니다. RESTfulAPI 는 앞서말한 REST의 규칙을 지키며 API 를 개발하는걸 뜻합니다. 간단한 예시를 들어드리자면, 구글이나 네이버를 잠깐 검색해봐도 주소창에는 뭔가의 규칙에 의해 나와있습니다. 규칙성은 없지만 구글검색엔진의 백엔드 API 네이밍 룰을 따르겠죠? 장점 : RESTful API는 그 자체만으로도 API의 목적이 쉽게 이해가 됩니다. 단점 : 위의 구글 예시와같이 표준규약이 없어서 안티패턴으로 작성되는 경우가 흔합니다....

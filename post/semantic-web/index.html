[HTML] Semantic Web, Semantic Tag 이 뭔가요?

Semantic Web은 의미를 정확히 부여하는 기술입니다.
December 27, 2021 · Me | Suggest Changes

Semantic Web 이란?#

Semantic Web은 HTML4 에서 HTML5 올라가면서 생기게 되었습니다.

의미를 정확히 부여하는 기술입니다.

기존에는 컴퓨터가 메타데이터를 해석하기만 했다면

Semantic Web은 사람의 눈으로 보기에 용이한 메타데이터와 자연어로 구성되어있습니다.

예를 들면,

<div id="header"></div>

이렇게 이 div 는 header 라고 나타내었던것을

<header></header>

태그에 header 라고 의미를 부여해 더 효율적으로 작성할수있습니다.

     

Semantic Tag 란?#

Semantic Tag 도 Semantic Web 과 마찬가지로 ‘의미가 있는’ 태그를 뜻합니다.

이걸 언제써야하는지는

img 태그와 css의 background-image 를 예시로 들면

<img id="my-img" alt="내 얼굴" src="/img/profile.png">

만약 /img/profile.png 의 사진의 경로가 잘못되어 브라우저가 불러오지 못하는 상황이 있을때 alt 태그로 이 사진은 얼굴 사진이라는것을 알 수 있습니다.

허나 css 에 background-image 를 사용하게되면

<div id="my-img"></div>

#my-img {
    background-image: /img/profile.png
}

단지 div 에 이미지를 추가 한 것일뿐 어떠한 정보나 메타데이터가 없습니다.

웹의 SEO 측면에서 봤을때도 메타데이터가 없는 후자의 CSS 방식은 불리할수밖에없습니다.

참고로 img 와 같은 Semantic Tag를 조금 소개해드리자면

<header>
<nav>
<footer>
<section>
<article>

등이 있습니다.

더 많은 정보는 W3C 를 참고 바랍니다.
